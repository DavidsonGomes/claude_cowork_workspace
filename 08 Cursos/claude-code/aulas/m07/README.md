# M07 — Capstone

## Objetivo do módulo
Ao final deste módulo, o aluno vai entregar um projeto completo ponta a ponta usando todos os critérios do curso: execução guiada, qualidade técnica, contexto gerenciado e operação sustentável.

## Aulas

| # | Aula | Duração | Formato | Status |
|---|------|---------|---------|--------|
| 1 | Briefing e escopo do projeto final | 20 min | vídeo + prática | draft |
| 2 | Implementação incremental | 30 min | prática | draft |
| 3 | Hardening e validação final | 20 min | prática | draft |
| 4 | Defesa técnica com evidências | 20 min | prática | draft |

## Entregável do módulo
Projeto funcional + documentação técnica + rubrica auto-preenchida com todas as evidências verificáveis.

## Critérios de aprovação
- [ ] Projeto funcional demonstrável por terceiros
- [ ] Testes passando + lint zerado + commits limpos
- [ ] Documentação essencial presente (decisões, arquitetura mínima)
- [ ] Rubrica preenchida com todos os critérios do curso atendidos
