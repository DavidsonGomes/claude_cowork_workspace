# Live 01 — Fichas de aulas (normalizado v1)

## Aula 01 — Introdução ao curso
- Objetivo: apresentar a proposta do curso, blocos da aula e visão de transformação da profissão.
- Conteúdo principal: nova era do software, engenharia agêntica em 3 classes, visão geral do método e demonstração prática.
- Exercício prático: escrever meta pessoal do curso (resultado em 30 dias) e 1 projeto-alvo para aplicar.
- Entregável verificável: documento curto com "objetivo + projeto + critério de sucesso".
- Checklist:
  - [ ] Entendeu os 5 blocos da aula
  - [ ] Definiu objetivo pessoal claro
  - [ ] Definiu 1 projeto para aplicar o método

## Aula 02 — A nova era do software
- Objetivo: entender a transição do modelo tradicional para execução com agentes.
- Conteúdo principal: gargalos do software tradicional, remoção de gargalo humano na execução e salto de produtividade com IA.
- Exercício prático: mapear processo atual de desenvolvimento e apontar 3 pontos de gargalo para agentificação.
- Entregável verificável: quadro "antes/depois" com tempo, custo e velocidade de entrega.
- Checklist:
  - [ ] Identificou gargalos do modelo atual
  - [ ] Descreveu fluxo alvo com agentes
  - [ ] Definiu métrica de ganho esperada

## Aula 03 — O que é um agente
- Objetivo: diferenciar IA conversacional de agente executor.
- Conteúdo principal: agente como IA com ferramentas/skills, execução de tarefas e automação.
- Exercício prático: listar 3 tarefas do seu dia que um agente pode executar além de responder chat.
- Entregável verificável: documento curto com "tarefa -> ferramenta necessária -> critério de sucesso".
- Checklist de conclusão:
  - [ ] Explicou diferença entre chat e agente
  - [ ] Identificou pelo menos 3 tarefas automatizáveis
  - [ ] Definiu 1 critério de qualidade por tarefa

## Aula 04 — Application Layer vs Agent Layer
- Objetivo: entender a arquitetura em camadas (software tradicional + camada agentic).
- Conteúdo principal: limites do software tradicional; agente como camada de decisão e execução.
- Exercício prático: desenhar fluxo "usuário -> agente -> aplicação/API -> banco" para um caso real.
- Entregável verificável: diagrama textual (ou imagem) da arquitetura-alvo.
- Checklist:
  - [ ] Identificou responsabilidades de cada camada
  - [ ] Mapeou ao menos 1 ferramenta externa usada pelo agente
  - [ ] Definiu onde ocorre decisão/autonomia

## Aula 05 — Papel do humano agora
- Objetivo: mudar foco de codar para orquestrar e validar.
- Conteúdo principal: engenheiro agentic classe 1 como controlador do fluxo.
- Exercício prático: transformar um objetivo de produto em plano de execução por agentes.
- Entregável verificável: plano com agentes envolvidos + validações humanas.
- Checklist:
  - [ ] Objetivo definido
  - [ ] Agentes/roles definidos
  - [ ] Pontos de validação humana definidos

## Aula 06 — Os 7 pilares
- Objetivo: dominar base da Classe 1.
- Conteúdo principal: identidade/contexto, planejamento, ferramentas/skills, loops de revisão, templates, chains/experts, aprendizado contínuo.
- Exercício prático: autoavaliação de maturidade (0-3) em cada pilar.
- Entregável verificável: scorecard dos 7 pilares + plano de melhoria.
- Checklist:
  - [ ] Avaliou os 7 pilares
  - [ ] Identificou 2 lacunas críticas
  - [ ] Definiu ação concreta por lacuna

## Aula 07 — Problema do vibe coding
- Objetivo: entender risco de construir sem método.
- Conteúdo principal: inconsistência, baixa escalabilidade, retrabalho e custo.
- Exercício prático: comparar 1 caso "vibe" vs "com spec".
- Entregável verificável: tabela com riscos, custo, previsibilidade e qualidade.
- Checklist:
  - [ ] Evidenciou falhas de abordagem sem método
  - [ ] Definiu critério mínimo para começar execução

## Aula 08 — Como Spec-Driven muda tudo
- Objetivo: consolidar princípio "plano é o novo código".
- Conteúdo principal: investimento pesado em planejamento para execução robusta.
- Exercício prático: escrever mini-spec com regras, escopo e restrições.
- Entregável verificável: mini-spec revisada por IA + ajustes manuais.
- Checklist:
  - [ ] Spec contém objetivo, regras e limites
  - [ ] Passou por revisão
  - [ ] Está pronta para execução

## Aula 09 — Introdução ao BMAD
- Objetivo: entender BMAD como framework operacional de agentes.
- Conteúdo principal: papéis (PM, arquiteto, dev, QA), fluxo de planning->build->review.
- Exercício prático: mapear quais agentes serão usados no projeto da turma.
- Entregável verificável: matriz "agente -> responsabilidade -> artefato".
- Checklist:
  - [ ] Papéis definidos
  - [ ] Responsabilidades sem sobreposição crítica
  - [ ] Artefatos por papel definidos

## Aula 10 — Conhecendo as ferramentas
- Objetivo: escolher stack mínima de trabalho.
- Conteúdo principal: IDEs, plataformas de geração, deploy e libs auxiliares.
- Exercício prático: decidir stack principal + stack backup com justificativa.
- Entregável verificável: decisão de stack (ferramenta, custo, uso, limite).
- Checklist:
  - [ ] Stack principal definida
  - [ ] Critério de troca definido
  - [ ] Custos e limites mapeados

## Aula 11 — Instalando BMAD
- Objetivo: preparar ambiente funcional de execução.
- Conteúdo principal: setup em pasta limpa/projeto existente, seleção de IDEs e idioma.
- Exercício prático: instalar BMAD e validar estrutura de arquivos gerada.
- Entregável verificável: print/registro da árvore inicial + confirmação de execução.
- Checklist:
  - [ ] Instalação concluída
  - [ ] Agentes e pastas de output presentes
  - [ ] Ambiente pronto para iniciar planejamento

## Aula 12 — Iniciando um projeto
- Objetivo: executar pipeline inicial de criação de produto com agentes.
- Conteúdo principal: briefing -> PRD -> arquitetura -> design -> épicos -> stories -> sprint status.
- Exercício prático: gerar artefatos do projeto exemplo e revisar granularidade de épicos/stories.
- Entregável verificável: pacote inicial de documentos + 1 story pronta para implementação.
- Checklist:
  - [ ] Briefing gerado e validado
  - [ ] PRD e arquitetura gerados
  - [ ] Épicos/stories criados com revisão
  - [ ] Sprint status iniciado
