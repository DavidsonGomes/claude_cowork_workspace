# M01 — Método e Especificação

## Objetivo do módulo
Ao final deste módulo, o aluno vai saber transformar um briefing em PRD mínimo, épicos e stories com critério de aceite, produzindo uma spec acionável antes de escrever qualquer linha de código.

## Aulas

| # | Aula | Duração | Formato | Status |
|---|------|---------|---------|--------|
| 1 | Por que Spec-Driven supera vibe coding | 15 min | vídeo | draft |
| 2 | Introdução ao BMAD — estrutura e instalação | 15 min | vídeo + prática | draft |
| 3 | Briefing → PRD → Arquitetura → Design | 20 min | vídeo + prática | draft |
| 4 | Épicos e stories com critério de aceite | 20 min | vídeo + prática | draft |

## Entregável do módulo
Spec mínima completa: briefing aprovado + PRD + épicos + stories com critério de aceite para o projeto do curso.

## Critérios de aprovação
- [ ] Stories são acionáveis e testáveis (cada uma tem critério de aceite verificável)
- [ ] PRD define escopo claro (o que entra / o que não entra)
- [ ] Épicos estão decompostos sem dependências circulares
