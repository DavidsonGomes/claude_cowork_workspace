# Classe 1 — Checklist Operacional Oficial

Use este checklist para validar execução mínima da Classe 1.

## 1) Planejamento
- [ ] Problema e objetivo definidos em 1 parágrafo
- [ ] Escopo da entrega delimitado (o que entra / o que não entra)
- [ ] Briefing aprovado

## 2) Especificação
- [ ] PRD mínimo criado
- [ ] Épicos definidos
- [ ] Stories quebradas com critério de aceite

## 3) Execução
- [ ] Implementação por story (sem pular etapas)
- [ ] Review técnico por outro agente/modelo ou checklist formal
- [ ] Ajustes aplicados após review

## 4) Versionamento e rastreabilidade
- [ ] Commit por unidade lógica (story/ajuste)
- [ ] Histórico legível (mensagens de commit claras)
- [ ] Evidência de rollback possível

## 5) Qualidade mínima
- [ ] Entregável funcional demonstrável
- [ ] Fluxo principal sem erro crítico
- [ ] Documento final com decisões e trade-offs

## Regra de aprovação
A Classe 1 só fecha quando todos os blocos acima estiverem completos.
