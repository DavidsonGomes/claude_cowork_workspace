# Roadmap — OpenClaw

## Fase atual
Planejamento reorganizado em 2 níveis separados: Core (single-agent) e Scale (multiagentes). Materiais de setup/instalação já produzidos e aproveitáveis. Foco atual: concluir Nível 1 ponta a ponta antes de iniciar Nível 2.

## Prioridades

### P0 — Crítico (esta semana)
- [ ] Fechar outcomes e critérios de aprovação por módulo do Nível 1 (M1..M7)
- [ ] Mapear cada aula do Core para um artefato real do stack OpenClaw (Build the Real Thing)

### P1 — Importante (próximas 2 semanas)
- [ ] Converter módulos do Nível 1 em aulas graváveis (15–25 min cada)
- [ ] Definir projeto transversal do Core (escopo, entregas e checkpoints)
- [ ] Padronizar rubrica de avaliação por aula do Nível 1

### P2 — Planejado (próximo mês)
- [ ] Fechar estrutura comercial: Core standalone + Scale avançado + bundle
- [ ] Preparar trilha Nível 2 (Scale) a partir do capstone do Core
- [ ] Fechar outcomes e critérios de aprovação por módulo do Nível 2

## Decisões pendentes
- Estratégia comercial: Core como produto independente vs pré-requisito obrigatório para Scale
- Projeto transversal do Core: escopo e critério de conclusão ainda não definidos

## Princípios pedagógicos
- Dominar single-agent antes de qualquer conteúdo de multiagente
- Cada módulo fecha em: problema real → modelo mental → execução → validação
- Estratégia Build the Real Thing: cada aula adiciona uma camada operacional do mesmo sistema
- Foco em operação real — não apenas teoria de prompts
- Não tratar multiagentes como pré-requisito do início
